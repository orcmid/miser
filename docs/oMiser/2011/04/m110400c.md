## PlaceHolder

This is the place for the MOb API.  It will describe the way that the initial
MOb is connected when oMiser is initiated and also specify the helpful
functions.

There are a couple of challenges.  One has to do with COM requiring Unicode
strings, and those of a particular organistation, BSTR.  These are poor for
hashing, and how that is reconciled will need to be worked out.  A positive
characteristic of BSTRs is that they don't have to be scanned to find the
end and length.  But since that is all done only at the creation of entries
for named obs, that would be tolerable.  It would also be tolerable to use
conventional \\0-terminated char\[\] strings using UTF8, for oMiser purposes
and we can deal with BSTR behind the scenes if it comes to that.
