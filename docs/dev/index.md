## 1. Pre-Alpha Status

There are incomplete and skeletal materials at this time.

Documentation generally precedes the operational status of associated software
components.

Development of early provisional software and associated tests is reflected
on the `devLab` folder of the Miser Project
[GitHub repository](https://github.com/orcmid/miser).

Activity is proceeding "inside-out" starting with stages of oMiser code and
followed by stages of oFrugal development that rely on then-available
oMiser. This spiraling will continue until there is enough operational
software to package and release in an alpha-level release.

The performance of alpha-level releases will not be optimal.  The alphas will
also be heavily instrumented and that will be a factor in experimental use
of the distributions.

Achievement of beta-level release will be when all functionality is complete
and the oFrugal application is a candidate for reliable usage.  Breaking
changes are not expected, but there may continue to be fixes and performance
improvements.

For examples of how packaging and delivery is patterned, distribution of
the [VCrayApp DevKit](https://orcmid.github.io/nfoTools/dev/D211101/) style
is employed, although not as a developer setup, as a command-line utility
instead.
