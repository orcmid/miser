Exemplar Customization

  1. hybridForm is a template and exemplar for repurposing as Miser Project topic sections with their own article subsections.  This current Article Diary & Job Jar page is the exemplar for hybridForm article folios.  The practice is to have the article hyymmnn page hyymmnna.htm always provide a Diary & Job Jar for the particular article.  The portable practice is to always have this page customized for that purpose.  Customization is as follows.
  2. In the top banner, the site/project image should already appear correct in the left cell.  The link should be to "../../../", the project home directory.
  3. In the title section of the top banner, the link "hymmnn" should be adjusted to the correct form for this topical article.  The link should be to the article anchor in the (transposed) h000001.htm catalog.  The "Topic Articles" link should be linked to the topic catalog, the transposed h000001.htm location.   The word Topic in the link should be replaced with the actual topic name.
  4. In the far right block of the top banner, without altering the links, change the topic name from to the name of the topic under which this article is located.  The link should remain "../../".
  5. In the far right block of the top banner, without altering the links, change the annual folder name for "yyyy" to the year in which articles are being (or have been) initiated.  Change the monthly folder name for "mm".  Then correct the name of this file (absent .htm) and correct the link accordingly..
  6. Adjust the Revision History to reflect the initialization of this page.  Retain the supplied 0.0.0 to reflect this exemplar origin.  For the transposed new version, note the format of revision-number anchors and the links from the version specified in the right block of the page-top banner.
  7. On the properties for this file, revise it to reflect the actual article is a page for.
  8. No changes should be required in the footer banner except for the "created" entry and possible identity and link to the author.  Verify that there are no mistaken links in that banner.
  9. When the customization is complete, delete this complete "Template Customization" section and commit this repurposed version to the repository.
  10. The formatting of table entries and the way they are moved to in-progress and done (or other disposition), along with colorization plus cut-and-past into the reverse-chronological bunching of closed items can be discerned by comparison with in-use forms of this page in other parts of the web site.
